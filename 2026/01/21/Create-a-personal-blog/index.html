使用 Hexo 和 GitHub 创建的个人博客

2026-01-21

安装 git 工具

可以通过 git官方安装，也可以使用scoop来安装

输入以下命令检查 git安装是否成功：

git -v

安装 nodejs

使用 Scoop安装nvm（node 包管理工具）

使用命令nvm install 「版本号」来安装nodejs，使用nvm use 「版本号」指定使用的nodejs，使用node -v查看 node 版本

安装 Hexo

使用 如下命令安装hexo

npm install hexo-cli -g

https://hexo.io/zh-cn/

测试hexo是否安装成功

安装 hexo 依赖

这一步可以留着到后面创建了博客目录后在博客目录下执行安装就行，无需全局安装。

使用 npm安装

npm install --save hexo-deployer-git

如果出现问题，可以先用 Scoop 安装 pnpm，使用如下命令来安装依赖：

pnpm add hexo-deployer-git

git 配置 SSH Key

为什么要配置SSH key？
目的：可以免密的将本地的源码和资源上传到github，无需要每次都输账号和密码

生成 ssh-key

输入生成命令后。连续回车三次即可

ssh-keygen -t rsa -C "GitHub邮件地址"

利用如下命令查看生成的 ssh-key

cat ~/.ssh/id_rsa.pub

配置 ssh-key 到 GitHub

配置账号和密码

git config --global user.name "FbyBruce" #你的github用户名
git config --global user.email "xxx@163.com" #填写你的github注册邮箱

测试连接

使用如下命令测试 GitHub 连接：

ssh -T git@github.com

如果出现下面的情况，

在C:\Users\用户名\.ssh 下新建一个 config 文件，写入如下内容：

Host github.com
  Hostname ssh.github.com
  Port 22

可以发现再次测试连接就可以连接成功了

搭建个人博客

heox的基本命令

hexo new "postName" #新建文章
hexo new page "pageName" #新建页面
hexo generate #生成静态页面至public目录
hexo server #开启预览访问端口（默认端口4000，'ctrl + c'关闭server）
hexo deploy #部署到GitHub
hexo help  # 查看帮助
hexo version  #查看Hexo的版本
----------------------------------------------------------------
# 可以使用首字母缩写
  - hexo g -> hexo generate
  ......

创建本地博客

在本地新建一个空的保存博客的存放目录，比如我放在了E:\PersonalBlog\mysite

进入到博客目录利用hexo init命令初始化个人博客

这里我创建一个新的测试文件展示：

生成静态网页

使用hexo g生成静态网页，使用 hexo s进行本地运行预览

部署到 GitHub

新建一个 GitHub 仓库，注意仓库名，设置为公开，并添加README文件。

注意：给仓库起名字必须是 用户名.github.io

编辑_config.yml 文件，_config.yml在博客存放目录下

deploy:
  type: git
  repo: git@github.com:FbyBruce/FbyBruce.github.io.git # 改为自己的仓库地址
  branch: main

使用hexo d命令部署到 GitHub

在浏览器访问FbyBruce.github.io

编写博客

比如新建一篇名为Create a personal blog的个人博客，可以看到创建成功之后给出的md文件地址

我们通过这个md文件去写博客即可。

使用hexo g生成 html文件，执行hexo s在本地运行进行测试

但是文件图片没有正确显示出来。

这是因为 Hexo 默认将所有文章资源（如图片）放在 source/ 根目录下，而没有按文章组织，导致路径在部署后失效。

从 Hexo 3.0+ 开始，官方推荐使用 post_asset_folder（文章资源文件夹） 功能来解决这个问题。

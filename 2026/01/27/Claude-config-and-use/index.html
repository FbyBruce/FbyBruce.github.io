Claude 安装

  1. Claude官网安装：

  1. 可以使用Scoop进行安装

安装之后在终端输入claude启动 Claude，但初次直接使用访问的是 Anthropic 的 Claude 模型，在国内很多人是没法访问这个模型的，会报下面的错误。

安装 cc-switch

为了解决上面的问题，我们可以安装这个 cc-switch，这个工具可以方便我们对Claude Code的模型 API 等进行配置。

可以从GitHub 官网安装，我这里同样使用Scoop安装

安装好之后就是下面这样一个工具，这里可以配置模型 API，MCP，Skills 等

点击右上角的➕，选择模型供应商，配置你的 API。我这里使用阿里云百炼配置QWen-Coder

重启终端再次尝试通过claude启动 Claude，可能还是会报上面同样的无法访问的错误，我们只需要在配置文件加上一行配置即可绕过地区限制（即修改**.claude.json**文件）

在C:\Users\你的用户名目录下找到**.claude.json**文件，添加如下配置：

"hasCompletedOnboarding": true

再次尝试即可正常启动

简单实战

通过 Claude 打开运行这个 index.html 文件，在终端中输入!，终端就处在bash模式下了，输入start index.html（Windows）来运行这个 html，macOS 中利用open index.html

效果展示

下面改为前端流行架构，按shift + Tab切换为 plan mode 模式，让 Claude 在执行之前先帮我们做计划

如果我们有新的要求，可以选择 4 并输入要求

后续可能还会让我们选择模式，如果你希望每次执行任务前都亲自检查，可以选择每次都询问，或者直接选择自动接受，则不再询问。

但后续需要 Claude 执行终端命令时还是会询问，我们可以在开启 Claude 时加一个参数--dangerously-skip-permissions，这样 Claude 就可以随意执行终端命令不再询问，但存在安全隐患，不建议开启。

在 Claude 中还有 MCP、Skills 等各种使用姿势，后期我会逐渐补充~
